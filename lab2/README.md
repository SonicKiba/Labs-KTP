# Лабораторная работа №2: Объектно-ориентированное программирование

## Вариант 8: Иерархия классов «Мебель»

## Описание
В работе реализована иерархия классов на тему мебели с использованием основных принципов ООП: абстракции, инкапсуляции, наследования и полиморфизма.

---

## Классы

### Furniture (абстрактный базовый класс)
- **Поля (private):** материал, цена, цвет.
- **Статическое поле:** `objectCount` — счётчик созданных объектов.
- **Конструкторы:** по умолчанию и с параметрами (материал, цена, цвет).
- **Геттеры и сеттеры** для всех полей.
- **Абстрактный метод:** `use()` — описывает назначение мебели.
- **Метод `showInfo()`** — выводит общую информацию (переопределяется в наследниках).
- **Статический метод `showObjectCount()`** — выводит количество созданных объектов.

### Table (Стол) — наследует Furniture
- **Дополнительные поля:** `seats` (количество мест), `shape` (форма стола).
- **Конструкторы:** по умолчанию (4 места, прямоугольный) и с параметрами.
- **Переопределённые методы:** `use()` («За этим столом можно работать или обедать»), `showInfo()` (добавляет места и форму).

### Chair (Стул) — наследует Furniture
- **Дополнительные поля:** `hasBackrest` (наличие спинки), `maxWeight` (максимальная нагрузка в кг).
- **Конструкторы:** по умолчанию (спинка есть, 120 кг) и с параметрами.
- **Переопределённые методы:** `use()` («На этом стуле можно сидеть»), `showInfo()` (добавляет спинку и вес).

### Bed (Кровать) — наследует Furniture
- **Дополнительные поля:** `width` (ширина, см), `length` (длина, см), `type` (тип кровати: односпальная, двуспальная и т.д.).
- **Конструкторы:** по умолчанию (90×200, односпальная) и с параметрами.
- **Переопределённые методы:** `use()` («На этой кровати можно спать»), `showInfo()` (добавляет размер и тип).

---

## Демонстрация в Main.java

- Создаются объекты: стол (дерево, 12500 руб., коричневый, 6 мест, прямоугольный), стул (пластик, 2500 руб., чёрный, спинка есть, 150 кг), кровать (дерево, 28900 руб., белый, 180×200, двуспальная).
- Вызываются `showInfo()` и `use()` для каждого.
- Показана работа геттеров и сеттеров: изменение цены стола.
- Полиморфизм: массив типа `Furniture[]` хранит объекты разных наследников, при итерации вызываются переопределённые методы.
- Upcasting и `instanceof`: объект `Furniture` приводится к `Table` для доступа к специфическому методу.
- Статический счётчик: `Furniture.showObjectCount()`.
